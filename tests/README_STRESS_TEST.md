# APISTD 稳定性测试脚本

## 快速开始

### 1. 安装依赖

```bash
pip install requests
```

### 2. 运行测试

#### 基础用法（并发模式）
```bash
python tests/stress_test.py
```

这将向 `http://192.168.1.5:5000` 发送 1000 次并发请求。

#### 自定义参数

```bash
# 指定目标地址
python tests/stress_test.py --base-url http://localhost:5000

# 增加迭代次数和并发数
python tests/stress_test.py --iterations 10000 --workers 50

# 顺序模式（适合调试）
python tests/stress_test.py --mode sequential --iterations 100

# 添加请求间隔（避免压力过大）
python tests/stress_test.py --delay 0.1
```

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--base-url` | `http://192.168.1.5:5000` | 目标服务器地址 |
| `--iterations` | `1000` | 迭代次数（总请求数 = 端点数 × iterations） |
| `--workers` | `10` | 并发工作线程数（仅并发模式） |
| `--mode` | `concurrent` | 测试模式：`concurrent` 或 `sequential` |
| `--delay` | `0` | 请求间隔时间（秒） |

## 测试端点

默认测试以下端点：
- `/users/5` (主测试端点)
- `/users`
- `/users/1` 到 `/users/5`

可以在脚本的 `endpoints` 列表中添加更多端点。

## 稳定性评估标准

| 成功率 | 评级 | 说明 |
|--------|------|------|
| ≥ 99.9% | ✅ 优秀 | 系统非常稳定 |
| ≥ 99% | ✅ 良好 | 系统稳定 |
| ≥ 95% | ⚠️ 一般 | 有少量错误 |
| < 95% | ❌ 需改进 | 错误率较高 |

## 建议

### 生产环境测试
```bash
# 高并发长时间测试
python tests/stress_test.py --iterations 10000 --workers 100 --base-url http://your-server.com
```

### 开发环境测试
```bash
# 低并发快速测试
python tests/stress_test.py --iterations 100 --workers 5 --mode sequential
```

### 持续监控
```bash
# 每 10 秒执行一次，持续监控
while true; do
    python tests/stress_test.py --iterations 10 --workers 2
    sleep 10
done
```

## 注意事项

1. **确保目标服务器正在运行**
2. **根据服务器能力调整并发数**
3. **生产环境测试前请先评估影响**
4. **建议先在开发环境测试**
5. **长时间测试建议添加 delay 参数**

## 故障排查

### Connection Error
- 检查服务器是否运行
- 检查网络连接
- 确认地址和端口正确

### Timeout
- 增加 `--timeout` 参数（需要修改脚本）
- 减少并发数
- 检查服务器性能

### 高错误率
- 查看服务器日志
- 检查 API 端点是否正确
- 降低并发压力
